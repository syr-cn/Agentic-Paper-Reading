# MemAtlas: SYR's Memory Paper Reading

A practical, research-oriented repository for tracking, reading, and synthesizing memory-centric AI papers.

## What this repo optimizes for
- **Fast triage**: quickly decide what is worth deep reading
- **High-signal notes**: concise but technically useful summaries
- **Research transfer**: convert paper insights into actionable project ideas
- **Continuity**: preserve evolving taste and judgment over time

## Core Focus
1. Agentic memory systems
2. Long-context reasoning
3. Retrieval-augmented reasoning (RAR)
4. Multi-modal memory for agents
5. RL for tool-using/planning agents
6. Scientific literature understanding

## Repository Structure
- `PAPER_PREFERENCES.md` — paper selection taste and ranking criteria
- `papers/` — source captures, candidate pools, and metadata
- `notes/` — one-note-per-paper deep reading outputs
- `readinglist/` — prioritized queues (`now` / `later`) and scoring references
- `weekly/` — weekly synthesis, trends, and open questions

## Recommended Workflow
1. **Capture** candidates into `papers/` (arXiv / HF / venue lists)
2. **Triage** with quick scoring in `readinglist/`
3. **Deep read** selected papers with `notes/TEMPLATE.md`
4. **Synthesize weekly** insights in `weekly/`
5. **Update taste** in `PAPER_PREFERENCES.md` as priorities evolve

## Minimal quality bar for accepted papers
- Clear and non-trivial problem formulation
- Strong evidence (ablations, comparisons, error analysis)
- Insight transferability to ongoing projects
- Honest treatment of limitations

## Suggested naming conventions
- Notes: `notes/YYYY-MM-DD_short-title.md`
- Daily capture: `papers/YYYY-MM-DD.md`
- Weekly synthesis: `weekly/YYYY-Www.md`

## Long-term goal
Build a high-quality "research memory" for fast retrieval, better judgment, and stronger paper-to-project conversion.
