## 1 一句话 Why-read
- **Key claim/contribution + key observation：** 将 skill library 作为 RL 训练中的内生状态（而非外挂检索器），让“选 skill—解题—写 skill”三步进入同一策略闭环，显著提升 OOD 数学推理表现。

## 2 CRGP
### C — Context
- RLVR/GRPO 系方法已成为数学推理后训练主流，但每题独立优化，经验难累计。

### R — Related work
- Reflexion / ExpeL / SimpleMem：主要是推理时检索，memory 与 policy 学习解耦。
- EvolveR / SkillRL / SAGE：开始引入 skill 库，但 skill 选择仍常由外部检索器主导。

### G — Research gap
- 现有方法通常不能让 policy gradient 直接塑造 skill selection，且 skill 库更新与 RL 目标弱耦合。

### P — Proposal
- 提出 ARISE：Manager-Worker 分层策略（共享参数）+ 两层 skill 库（cache/reservoir）+ 分层奖励（0/1/2）+ 两阶段训练（warm-up → skill-augmented）。

## 5 Why it matters for our work
- 与我们的 agent memory 主线高度一致：把 memory retrieval 从外挂模块推进到策略学习内部。
- 可直接映射到我们的多模态/长时程任务：Manager 做记忆调度，Worker 做任务执行。
