## 1) 一句话 Why-read（必填）
- **Key claim/contribution + key observation：** ARISE 把“选技能（before）—解题（during）—写技能（after）”统一到同一共享策略里，让 policy gradient 直接优化 skill selection，而不是把检索器外挂在 RL 之外。

## 2) CRGP 拆解 Introduction（必填）
### C — Context
- RLVR/GRPO 在数学推理上已验证有效，但常见训练范式是“每题独立优化、经验不沉淀”。
- memory/skill 方法虽能提升迁移，但很多仍把检索与更新放在策略外部。

### R — Related work
- **RLVR 线：** GRPO、Dr.GRPO、DAPO、GSPO。  
- **Memory/Skill 线：** Reflexion / ExpeL / SimpleMem（多为推理时注入）；EvolveR / SkillRL / SAGE（开始走训练环内 skill 化）。

### G — Gap
1. skill selection 常由外部 retriever 处理，policy gradient 难以端到端塑造“选哪个 skill”。  
2. skill 库更新常与 RL 目标弱耦合，难形成“策略提升 ↔ 记忆质量提升”的正反馈。

### P — Proposal
- **共享参数的 Manager-Worker**：同一策略既负责技能选择也负责解题生成。  
- **两层库（cache/reservoir）+ 五操作**：Add / Update / Evict / Load / Delete。  
- **分层奖励**：\(R\in\{r_0,r_1,r_2\}\)，实验设定 **r0=0, r1=1, r2=2**。  
- **两阶段训练**：Phase I warm-up（前 **Nw=500** steps）→ Phase II skill-augmented（step 501+）。  
- 每个 query 的 rollout 组大小 **G=8**，另有技能生成 rollout \(O_{G+1}\)。

## 4) Experiments（必须含具体数字）
### 4.1 Experimental setup（带数字）
- **训练数据：** DeepScaleR，约 **40K** 数学题（AMC/AIME/MATH/OlympiadBench）。  
- **模型：** Qwen3-4B-Instruct-2507、Phi-4-mini-instruct-3.8B。  
- **训练步数：** **2000** optimization steps。  
- **GRPO 参数：**
  - group size **G=8**；
  - per-device batch size **4**；gradient accumulation **2**；
  - effective batch **64 queries/update**；
  - AdamW，lr **1e-6**，weight decay **0.01**；
  - cosine schedule + **50** warmup steps；
  - max generation length **4096 tokens**（含 \(O_1...O_G\) 与 \(O_{G+1}\)）；
  - clipping \(\epsilon_c=0.2\)。
- **两阶段：**
  - Phase I：前 **500** steps，用 binary reward \(r^{task}\in\{0,1\}\)；
  - Phase II：step **501+**，改为分层奖励 \(R\in\{0,1,2\}\)。
- **评测：** 7 benchmarks（AMC23、AIME24、AIME25 + Omni-MATH 四子域），**Pass@1 平均 32 runs**。

- **缺失标注（复现相关）：**
  - A100-80GB 的**具体卡数未明确披露**；
  - 训练总 token / wall-clock / 单次训练成本：**未系统披露**；
  - 图1中 token overhead 的精确数值表：**正文未给完整表格**（主要通过图展示）。

### 4.3 Analysis experiments（强制“现象+解释”）
- **现象1：** 去掉分层奖励（退回 binary reward）时，性能下降最明显，且 skill utilization 从 **73% 降到 31%**。  
  **解释（作者）：** \(r_2>r_1>r_0\) 让“答对且有效用 skill”与“答对但没用 skill”在 advantage 上可区分，策略才会学会稳定调用有用技能。  
  **【标注】（我的判断）：** 这证明“记忆使用奖励”是内生化 memory 的关键杠杆，不是可有可无的装饰项。

- **现象2：** 随机注入 skill 会把 utilization 拉高，但准确率下降。  
  **解释（作者）：** 高调用率不等于高质量匹配，不相关 skill 反而污染上下文。  
  **【标注】（我的判断）：** 对 agent memory 设计的直接启发：要联合优化“调用率 + 匹配质量”，不能只盯前者。

- **现象3：** 去掉 \(O_{G+1}\) 后，skill 库停在 **24 seed skills**，OOD（Omni-MATH）退化最明显。  
  **解释（作者）：** 静态库无法覆盖 Olympiad 级题型变化，需要在线生成新 skill。  
  **【标注】（我的判断）：** 长生命周期 agent 同样如此：只靠冷启动记忆，后期会被分布漂移击穿。

- **现象4：** 去掉 confidence gate 对最终准确率影响较小，但 utilization 飙到 **91%**。  
  **解释（作者）：** gate 主要过滤边界样本噪声，抑制“过度调用”。  
  **【标注】（我的判断）：** gate 可视作 memory precision 控制阀，先控精度再谈召回。

## 5) Why it matters for our work
- **agent memory：** ARISE 给出“memory 被策略梯度直接优化”的闭环范式。  
- **long-context：** cache/reservoir 对应“工作记忆 + 长期记忆”分层治理，适合持续训练。  
- **multimodal/tool-agent：** Manager-Worker 可以扩展到跨模态 skill 路由（先选视觉/文本/工具 skill，再执行）。

### Case 对齐（>=2）
- **Case A（对齐 `2026-03-13_dive.md`）：** 两者都强调“结构化经验比单纯样本量更重要”；DIVE侧重数据合成结构多样性，ARISE侧重训练中 skill 结构内生化。  
- **Case B（对齐 `2026-03-12_retroagent.md`）：** 都强调经验可复用，但 RetroAgent 偏外部检索增强，ARISE进一步把检索/更新纳入同一策略优化目标。  
- **Case C（对齐 `2026-03-13_reasoning-judge.md`）：** 都重视可验证信号；reasoning-judge偏评估质量判别，ARISE把可验证回报直接转成可学习的分层奖励。
