## 1) 一句话 Why-read（必填）
- **Key claim/contribution + key observation：** TRT 提出一个无需外部反馈的 test-time 迭代自改进框架：模型在每轮生成多个 rollout（条件化于累积知识 + rollout-specific 策略），自我排序选出最佳，然后通过对比分析提炼"什么不该做"的知识列表，递归指导下一轮探索。关键发现：开源模型在 AIME-25/24 上达到 **100% 准确率**；o4-mini/o3 在 LiveCodeBench Hard 上分别提升 **+10.4/+14.8 pp**，且知识列表仅占上下文 <1.5%。

## 2) CRGP 拆解 Introduction（必填）
### C — Context
- LLM 推理能力通过 RL + 可验证奖励快速提升（OpenAI o1、DeepSeek-R1），但这些范式依赖外部监督信号。
- 核心问题：模型能否在 test time 无 ground-truth reward 的情况下自我改进？

### R — Related work
- **Meta-RL 线：** RL² (Duan et al., 2016)、Learning to RL (Wang et al., 2016)、MRT (Qu et al., 2025b)、LaMeR (Jiang et al., 2025a)——需要权重更新和复杂奖励校准。
- **Context engineering / self-improvement 线：** Reflexion、Self-Refine、ACE (Zhang et al., 2025)、Markovian Thinking——依赖外部反馈或仅做单 pass refinement。
- **Parallel scaling 线：** Self-consistency (Wang et al., 2022)、ToT (Yao et al., 2023)、RSA (Venkatraman et al., 2025)——聚合最终答案但不提取可迁移的失败知识。

### G — Gap
1. 现有方法缺乏**递归机制**把跨轮改进有效地向前传递。
2. 有效的 test-time 自改进需要**同时解决两个互补问题**：(i) 高效多样地探索解空间；(ii) 无 ground truth 下可靠地选择正确答案。单独解决任一个都不够——探索无验证产生噪声，验证无探索导致停滞。

### P — Proposal
- **Test-time Recursive Thinking (TRT)**：每轮三阶段循环——Generate（条件化于知识 + 策略生成 K 个 rollout）→ Select（自我排序选最佳）→ Reflect（对比分析提炼失败洞察 + 设计新策略）。
- 知识表示为**负约束（"don'ts"）**，每轮可剪枝 1 条过时知识保持紧凑。
- 针对不同领域设计不同的 SELECT 机制：数学用 mutual exclusivity + 历史拒绝答案；代码用自生成测试 + execution。

## 4) Experiments（必须含具体数字）
### 4.1 Experimental setup
- **任务/数据：**
  - 数学推理：AIME 2025（30 题）、AIME 2024（30 题）
  - 代码生成：LiveCodeBench v6 Hard（**203** 题）
- **模型/agent 配置：**
  - AIME：gpt-oss-120b（128K context）、Qwen3-235B-A22B-Thinking（256K context）
  - LiveCodeBench：o4-mini (High)（200K context）、o3 (High)（200K context）
  - 小模型消融：gpt-oss-20B、Qwen3-30B、Qwen3-4B
- **对比基线：**
  - Parallel Thinking + Majority Vote（Majority@64）
  - Recursive Self-Aggregation (RSA)（population 2 × 8 iterations）
  - Markovian Thinking（32K / full context）
- **评测指标：** Accuracy、Cumulative Best、pass@k
- **TRT 配置：**
  - AIME：K=1 rollout/round × 64 rounds，rolling majority vote
  - LiveCodeBench：K=2 rollouts/round × 8 rounds，self-generated test execution

### 4.3 Analysis experiments（强制"现象+解释"）

- **现象1：** 知识列表保持极度紧凑——数学任务 64 轮后 <1.5% context（gpt-oss-120b 平均 1.00%，Qwen3-235B 平均 0.47%）；代码任务 8 轮后 <0.35%（o4-mini 平均 0.16%，o3 平均 0.09%）。
  **解释（作者）：** 只存储蒸馏后的高层洞察（"don'ts"）而非完整 CoT/解法轨迹，实现高压缩比。
  **【标注】（我的判断）：** 这是 TRT 可扩展到长 horizon 的关键：记忆系统的价值不在量大，而在信息密度高。对 agent memory 设计直接启发——存"教训"比存"过程"高效得多。

- **现象2：** 记录失败（"Don'ts"）比记录成功（"Dos"）产生更强的性能增益（Fig. 8，AIME-25 上 Don'ts 始终优于 Dos）。
  **解释（作者）：** 负约束更有效地收缩搜索空间，避免重复走错路。
  **【标注】（我的判断）：** 这与 Reflexion 的 verbal RL 思路一致，也呼应 ARISE 的分层奖励设计——"什么不该做"是比"什么该做"更强的学习信号。对 agentic memory 的直接启发：failure memory 的优先级应高于 success memory。

- **现象3：** 在 203 道 LiveCodeBench 题中，有 **36 道（17.7%）** 是"breakthrough"——baseline 在 pass@10 下都没解出，但 TRT 通过迭代知识累积解出。
  **解释（作者）：** 策略性探索找到了随机采样在实际 budget 内无法触及的质性不同的解法。
  **【标注】（我的判断）：** 这说明"多样性 + 知识引导"的组合能突破纯并行采样的上限，对 agent memory 中的 skill/strategy evolution 有直接参考价值。

- **现象4：** Depth beats breadth——K=2/4/8 在足够轮数下收敛到相似的 Cumulative Best（78–82%）（Fig. 6）。
  **解释（作者）：** 如果增加并行 rollout 无法提升上界，则无需为 breadth 付出额外成本；迭代 refinement 跨轮才是主要增益来源。
  **【标注】（我的判断）：** 对 test-time compute scaling 策略的重要指导：在有限 budget 下优先投 depth（轮数）而非 width（每轮样本数）。

- **现象5：** 策略在失败后更频繁切换（82%）vs 成功后（74%）（Fig. 5b）；探索深度（unique technique clusters）与解题率正相关（Fig. 5c）。
  **解释（作者）：** 模型能"感知"自身进展并相应调整探索策略；策略多样性改善结果。
  **【标注】（我的判断）：** 自适应策略切换是 effective exploration 的核心，类似 agent 中的 meta-policy。

- **现象6：** 与 Markovian Thinking 对比（Fig. A14），后者在 full context 下性能反而降到 ~93% 并停滞，因为累积的 CoT 稀释了推理信号；TRT 通过压缩知识避免了这一问题。
  **解释（作者）：** 结构化知识提取（压缩到 <1.5% context）比直接拼接历史答案更有效。
  **【标注】（我的判断）：** 对 long-context agent 的核心教训：raw context accumulation 会导致 signal dilution，必须做 knowledge compression/distillation。

## 5) Why it matters for our work
- **Agentic memory：** TRT 的"知识列表 = 负约束蒸馏"范式直接对应 agent memory 中的"经验教训库"。核心洞察——失败知识比成功知识更有效——可迁移到 ReMemR1 / MemOCR 的 memory 写入策略。
- **Long-context reasoning：** 知识列表仅占 <1.5% context 却持续驱动改进，证明"压缩后的结构化记忆" >> "raw trajectory 拼接"。这与 Evo-Memory 的 Refine 机制和 ARISE 的 skill 精炼思路一致。
- **Test-time compute：** "Depth > Breadth" 结论对 inference budget 分配有直接指导意义。
- **Multi-modal agent + RL：** TRT 的 Generate-Select-Reflect 循环可以扩展到多模态 agent：视觉任务用 execution-based verification，文本任务用 mutual exclusivity，形成统一的 self-improvement loop。

### Case 对齐（>=2）
- **Case A（对齐 `2026-03-19_arise.md`）：** 两者都强调"知识/经验的迭代演化"是提升关键，但 ARISE 通过 RL 训练内化 skill，TRT 纯 test-time 不更新权重。互补视角：train-time skill evolution + test-time knowledge accumulation。
- **Case B（对齐 `2026-03-11_evo-memory.md`）：** Evo-Memory 强调 memory 的 Search-Synthesis-Evolve 流程，TRT 的 Generate-Select-Reflect 是其 test-time 对应物。两者都发现"记忆质量 > 记忆数量"。
