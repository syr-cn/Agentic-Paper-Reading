## 1) 一句话 Why-read
- **Key claim/contribution + key observation：** MIA 提出 Manager-Planner-Executor 三层架构，将 agent memory 分为 non-parametric（压缩历史轨迹）和 parametric（训练 Planner 内化经验）两种形式，通过 alternating RL + test-time learning 实现 memory 的双向转换与持续进化。核心发现：(1) 传统 long-context memory（RAG/Mem0/A-Mem）普遍低于 No Memory baseline，验证了长记忆上下文引入噪声的假说；(2) 7B Executor + MIA 平均提升 31%，超越 Qwen2.5-VL-32B 达 18%；(3) 无监督设定下 MIA 通过 peer-review 式 self-judgment 实现持续自进化。

## 2) CRGP 拆解 Introduction
### C — Context
- Deep Research Agents (DRAs) 通过 LLM + external tools 完成复杂开放任务，memory 系统是其积累经验和持续进化的关键。
- 现有 long-context memory 方法有四大硬伤：(1) 长上下文稀释注意力；(2) 无关内容引入噪声；(3) 存储开销随时间爆炸；(4) 大规模检索带来计算低效。
- 更根本的问题：long-context memory 只捕获 knowledge-oriented memory（事实/文档），而 DRA 需要 process-oriented memory（搜索轨迹、失败经验、成功策略）。

### R — Related work
- **Deep Research Agents：** DeepResearcher、Search-R1 用 RL 增强多轮搜索但限于纯文本；MMSearch-R1、DeepMMSearch-R1 扩展到多模态搜索但不利用历史经验。
- **Agent Memory：** ReasoningBank/MemoryBank 通过扩展 memory 增强推理；ExpeL 从成功/失败经验中学习；Mem-α/Memory-R1 用 RL 建模 memory 为 MDP；Agentic Memory/A-Mem 提出长短期/图式 memory 管理；Memento 冻结 LLM 微调 memory module；MemEvolve/Evo-Memory 探索 memory 动态演化。
- 但所有方法都存在：memory efficiency 低、结果不稳定、可解释性差，且无监督环境下的自主进化未被探索。

### G — Research gap
1. Planner 无任务特定训练，规划能力次优。
2. Few-shot 选择仅基于相似度，忽略质量、频率等维度。
3. Executor 无法充分理解和执行规划指令。
4. 本质问题：incompetent Planner + bloated memory + unprepared Executor = memory 系统收益有限。
5. 无监督环境下的 memory 自主进化完全空白。

### P — Proposal
- **MIA (Memory Intelligence Agent)：** Manager-Planner-Executor 三层架构。
  - **Memory Manager：** 非参数化 memory，压缩历史搜索轨迹为结构化 workflow。
  - **Planner：** 参数化 memory agent，基于检索到的历史经验生成搜索计划。
  - **Executor：** 按计划执行搜索和分析的 agent。
- 四大技术创新：
  1. Alternating RL 训练 Planner-Executor 协同。
  2. Test-time learning 让 Planner 在推理中持续进化。
  3. 参数/非参数 memory 双向转换实现高效 memory 进化。
  4. Reflection + unsupervised peer-review judgment 支撑开放世界自进化。

## 4) Experiments
### 4.1 Experimental setup
- **任务/数据：**
  - 多模态：FVQA-test, InfoSeek, SimpleVQA, LiveVQA, MMSearch, In-house 1/2（共 7 个 benchmark）
  - 纯文本：HotpotQA, 2WikiMultiHopQA, SimpleQA, GAIA-Text（共 4 个 benchmark）
- **模型/agent 配置：**
  - Executor：Qwen2.5-VL-7B（GRPO 训练，8 GPU，FVQA-train）
  - Planner：Qwen3-8B（GRPO 训练，4 GPU，FVQA-train + MATPO）
  - Memory Manager：Qwen3-32B（frozen）
  - LLM Judger：Qwen3-32B
  - 外部工具：local wiki25（text search）+ Serper（image search + online text search）
- **对比基线：**
  - Closed-source：GPT-4o, GPT-5.4, Gemini-2.5-Pro, Gemini-3-Flash
  - Search Agent：Qwen2.5-VL-7B/32B+ReACT, MMSearch-R1, DeepMMSearch-R1, WebWatcher, Deepeyes2
  - Memory-based：No Memory, RAG, Mem0, A-Mem, ReasoningBank, ExpeL, Memento
- **评测指标：** Qwen3-32B LLM Judger 判定答案正确率

### 4.3 Analysis experiments

- **现象：传统 contextual memory 方法（RAG/Mem0/A-Mem）普遍低于 No Memory baseline（多模态平均 -1.7/-7.0/-12.4）**
  **解释（作者）：** Long memory contexts 引入噪声，稀释模型对当前问题的注意力，导致推理能力退化。这是 long-context memory 应用于 DRA 的根本性缺陷。
  **【标注】这个发现直接呼应了我们在 ReMemR1 中观察到的"raw trace 无效"现象。Memory 不是越多越好，关键在于信息密度和相关性。MIA 通过 Manager 压缩 + Planner 抽象的两步过滤解决了这个问题。**

- **现象：Ablation 中"Only Memory"配置（直接将 memory 喂给 Executor）多模态平均 accuracy 下降 0.4，但"Memory for Planner"（memory 仅给 Planner 做规划参考）提升 3.5**
  **解释（作者）：** Memory 应当作为 Planner 的规划先验，而非直接注入 Executor 的 context。Planner 作为认知中枢过滤和抽象 memory，消除噪声后以 plan 形式传递给 Executor。
  **【标注】这是本文最重要的架构洞察。它说明 memory 的有效使用需要一个"认知中间层"——不是让执行者直接看历史，而是让规划者从历史中提炼策略。这对 agentic memory 系统设计有范式级启发。**

- **现象：Unsupervised MIA 在纯文本 benchmark 上超越几乎所有 supervised baselines，仅略低于 supervised MIA；多 epoch 自进化持续提升（FVQA: 65.1→66.4→67.1；2Wiki: 71.6→73.4→74.7）**
  **解释（作者）：** Peer-review 式 unsupervised judgment（3 Reviewer + 1 AC 的多维度正交评估）能有效替代 ground-truth 作为 supervision 信号。通过 Dimensional Orthogonality（逻辑/格式/事实分离）、Evidence-Based Accountability（要求引用证据）和 Conflict Resolution via Meta-Decision（AC 识别致命缺陷），避免了单一 LLM-as-judge 的"幻觉客观性"问题。
  **【标注】这个无监督自进化机制是本文最有前瞻性的贡献。它不依赖 ground-truth，意味着 MIA 可以在真实的开放世界中持续学习——这是大多数 agentic memory 系统做不到的。Peer-review 框架的设计也很巧妙：把评估任务分解为正交维度后，单个 LLM 的 judgment 质量显著提升。**

- **现象：MIA 对 GPT-5.4 的增益最大（+8.9/+6.4），而 Claude-Sonnet-4.6 增益最小（+1.8/+1.7），增益与 Executor 基础能力负相关**
  **解释（作者）：** 更强的 Executor 本身已经内化了部分搜索策略，MIA Planner 的规划指导的边际增益相对较小。但即便对最强模型，MIA 仍能提供一致的正向增益。
  **【标注】这个 scalability 结果很有意思，但解释有些反直觉——GPT-5.4 应该比 Claude-Sonnet-4.6 更强。可能的真实解释是：GPT-5.4 的 ReAct 模式 compliance 更好（更严格遵循 plan），因此从 Planner 的规划中获益更多。**

- **现象：训练分析中 Planner 的 reward 曲线上升比 Executor 慢得多，response length 收敛也更慢**
  **解释（作者）：** Planner 的动作通过 Executor 间接影响结果，reward 信号不稳定且有延迟。TTL 阶段 response length 自动适应数据集特征（2Wiki 变短、LiveVQA 变长），说明 RL 有效捕获了不同数据集的推理模式。

## 5) Why it matters for our work
- **直接相关：** MIA 引用了我们的 ReMemR1 工作（Shi et al., 2025），并将其置于 long-context memory 的局限性讨论中。MIA 的 Manager-Planner-Executor 架构可以视为对 ReMemR1 "revisitable memory" 思路的进一步发展——从被动的 memory revisit 到主动的 memory planning + evolution。
- **架构启发：** Memory 应作为 Planner 的输入而非 Executor 的 context，这是一个值得在我们的 agentic memory 系统中验证的设计原则。
- **Alternating RL 训练：** 两阶段交替训练 Planner 和 Executor 的范式，对我们的多模态 agentic RL 项目有直接参考价值——可以将 memory module 和 reasoning module 解耦训练。
- **无监督自进化：** Peer-review 式 self-judgment 是一个可复用的技术组件，可以直接应用于任何需要无监督 reward 信号的 agent training pipeline。
- **Test-time learning：** MIA 证明了在推理过程中同步更新 Planner 参数的可行性，这对 lifelong learning agent 的设计有重要启发。

## 6) Actionable next step
- [ ] 对比 MIA 的 "memory → Planner → plan → Executor" 流程与 ReMemR1 的 "revisitable memory → reasoning" 流程，量化 planning abstraction 层带来的增益
- [ ] 将 peer-review 式 unsupervised judgment 机制移植到我们的 RL training pipeline 中，替代 ground-truth reward
- [ ] 探索 alternating RL 训练范式在 multi-modal agent 上的适用性：将 memory/planning module 与 execution module 解耦训练
- [ ] 在 Evo-Memory benchmark 上对比 MIA 的 bidirectional memory conversion 与其他 memory evolution 方法
- [ ] 研究 MIA 的 memory compression 策略（轨迹→workflow summary）对 ReMemR1 的 memory 格式设计的启发

## 7) 评分解释
- **质量分 2/2：** 方法设计系统完整（Manager-Planner-Executor + alternating RL + TTL + unsupervised evolution），实验覆盖 11 个 benchmark（7 多模态 + 4 纯文本），baseline 对比全面（closed-source + open-source + memory-based 共 15+ 方法），ablation 逐模块验证有说服力，closed-source executor 泛化实验展示了 scalability。
- **Observation 分 1/2：** "Contextual memory 普遍低于 No Memory" 是有价值的负面发现，"memory → Planner 优于 memory → Executor" 是有架构指导意义的 insight。但核心思路（将 memory 转化为 plan 再指导执行）在 Memento 和 ExpeL 中已有雏形；unsupervised peer-review judgment 虽新颖但评估不够充分（仅对比了无监督 vs 有监督 MIA，未与其他 self-judgment 方法对比）。此外，多个关键实验（In-house 1/2）的数据集不公开，可复现性受限。
- **总分 4/5：** 基础 1 + 质量 2 + Observation 1 = 4
- **为什么不是更高分：** (1) Planner 训练依赖 Qwen3-32B 作 Judger + Manager，实际部署成本未讨论；(2) In-house 数据集不公开；(3) 无监督评估框架虽有创意但缺乏与其他 self-judgment 方法的对比；(4) memory compression（轨迹→workflow）由 frozen Qwen3-32B 完成，压缩质量的 failure case 分析缺失；(5) TTL 的 online learning 需要多次 rollout + Executor 交互，推理时间成本的分析不充分。
