## 1) 一句话 Why-read
- **Key claim/contribution + key observation：** GEMS 把 text-to-image 生成重构为 agentic 迭代优化问题，通过 Agent Loop（闭环迭代）+ Agent Memory（分层压缩轨迹记忆）+ Agent Skill（可插拔领域技能），让 6B 轻量模型 Z-Image-Turbo 在 GenEval2 上超过 SOTA Nano Banana 2。核心 observation：raw reasoning traces 直接塞进记忆反而无效，压缩成"经验摘要"才有用（+2.5 vs +0 on GenEval2）。

## 2) CRGP 拆解 Introduction
### C — Context
- 多模态生成模型在通用任务上进步显著，但在复杂指令遵循和专业下游任务上仍然挣扎。
- 推理时扩展（inference-time scaling）在 LLM 领域已证明有效，但在图像生成中的应用还处于早期。

### R — Related work
- **Inference-time scaling 线：** Random Search、Maestro（pairwise comparison 自演化）、CRAFT（MLLM feedback + prompt 迭代更新）。
- **Agent 系统线：** Claude Code 等 agentic 框架在代码生成中验证了迭代优化的有效性。
- **局限：** Maestro 只关注当前/最佳状态，缺乏全局历史视角；CRAFT 无专门记忆管理；所有方法对下游特化任务支持弱。

### G — Gap
1. 现有迭代方法缺乏 persistent memory：要么只看上一步，要么只保留 best，无法利用完整优化轨迹。
2. 通用 prompt rewriting 在特定领域（如文字渲染）经常损坏关键信息，缺少领域知识注入机制。
3. Raw reasoning traces 冗余严重，直接塞进 context 反而增加噪声。

### P — Proposal
- **Agent Loop：** Planner → Decomposer → Generator → Verifier → Refiner 的闭环，最多 N_max=5 轮迭代，带 early stopping。
- **Agent Memory：** 分层压缩 — factual artifacts（prompt, image, feedback）原样保留，reasoning traces 用 Compressor 压缩为 Experiences。
- **Agent Skill：** 可扩展技能库，on-demand loading（只加载名称+描述，触发后才加载详细指令），支持 Creative Drawing / Aesthetic Drawing / Text Rendering / Spatial Intelligence 四种技能。

## 4) Experiments — Key Numbers

### Main Results
| Benchmark | Metric | GEMS | Best Baseline | Delta |
|-----------|--------|------|---------------|-------|
| GenEval2 | Overall | 63.5 | 60.7 (Nano Banana 2) | +2.8 |
| T2I-CompBench | Avg | +14.22 over base | — | — |
| Downstream (4 tasks) | Avg | +14.03 over base | — | — |

### Ablation (GenEval2)
- Base (Z-Image-Turbo): 31.0
- + Agent Loop: 52.4 (+21.4)
- + Agent Memory: 61.4 (+9.0)
- + Agent Skill: 63.5 (+2.1)

### Memory Ablation (GenEval2 delta over Loop-only)
- No memory: baseline
- + compressed experiences: +2.5
- + raw thoughts: +0.0 (noise!)
- Best config = compressed experiences only

### Efficiency
- Early stopping reduces avg iterations: 3.26 to 2.80
- Skills trigger on-demand, no wasted context on irrelevant domains

### Limitations
- 依赖外部 MLLM（GPT-4o）做 Verifier/Refiner，成本高
- 迭代轮数仍然需要手动设 N_max
- Skill 库目前只有 4 个，扩展性待验证
## 5) Why it matters — 对我研究的启发

1. **Memory 压缩 > Raw traces：** 这和 ReMemR1 的核心思想一致——agent 记忆需要 selective compression，不是把所有东西都塞进 context。GEMS 的消融（compressed +2.5 vs raw +0）给了一个干净的实验证据。
2. **Agentic generation 是 inference-time scaling 的自然延伸：** 从 Best-of-N → iterative refinement → full agent loop with memory，这条路径越来越清晰。GEMS 把 agent memory/skill 的概念引入生成领域，打通了 agent reasoning 和 generation 的界限。
3. **Skill 的 progressive exposure 设计：** 只加载名称+描述，触发后才加载详情——这对长 context agent 系统是一个实用的工程模式，值得在 MemOCR 后续工作中借鉴。
4. **局限也是机会：** 依赖外部 MLLM 做 judge 的成本问题，如果能用轻量 reward model 替代，就能大幅降低推理成本——这正是 RL-based refinement 的方向。
