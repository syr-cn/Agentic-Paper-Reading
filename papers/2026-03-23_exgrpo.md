## 1) 一句话 Why-read（必填）
- **Key claim/contribution + key observation：** ExGRPO 是首个系统研究"什么样的推理经验对 RLVR 最有价值"的工作：发现中等难度题目 + 低熵轨迹是最优经验来源，据此设计 bucketed replay buffer + entropy-based trajectory selection + mixed-policy objective，在 5 个骨干模型上平均提升 +3.5/+7.6（ID/OOD），并能稳定 on-policy 崩溃的弱模型训练。

## 2) CRGP 拆解 Introduction（必填）
### C — Context
- RLVR（Reinforcement Learning with Verifiable Rewards）已成为提升 LLM 推理能力的主流范式（DeepSeek-R1、Dr.GRPO 等）。
- 标准 on-policy 训练在每次梯度更新后丢弃 rollout 经验，计算效率低且不稳定。

### R — Related work
- **RLVR 线：** GRPO / Dr.GRPO / PRIME-Zero / Oat-Zero / GPG-Zero / RePO-Zero。
- **Off-policy 线：** LUFFY（off-policy PG + expert data）、SFT+RL、知识蒸馏。
- **Experience replay for LRM：** ReMix（off-policy replay）、RePO（online replay）、RLEP（well-trained policy replay）、RRL（retrospective replay）、ARPO（GUI agent replay）。

### G — Gap
1. 现有 experience replay 方法未系统研究"经验质量"对 RLVR 学习动态的影响。
2. 大多数 replay 方法忽略了 importance weighting 以纠正 off-policy 分布偏移。
3. 不加选择的 replay 可能引入"lucky hit"（答对但推理错误）的高熵轨迹，导致 snowball effect（累积错误模式）。

### P — Proposal
- **经验价值分析：** 发现 rollout correctness（question 维度）和 trajectory entropy（trajectory 维度）是经验价值的有效在线代理指标。
- **ExGRPO 框架：**
  1. **Experience Collection**：收集成功轨迹存入 replay buffer ℰ。
  2. **Experience Partition**：按 correctness rate 分桶（Easy/Medium/Hard）+ Retired Set。
  3. **Experience Selection**：按高斯 N(0.5, 1) 优先采 Medium 桶 + 选最低熵轨迹。
  4. **Mixed-policy objective**：(1-ρ)·on-policy + ρ·off-policy，ρ=50%。
  5. **Policy Shaping**：f(x) = x/(x+β) 非线性变换抑制高概率信号、放大新颖信号。
  6. **Delayed Start**：batch Pass@1 > 35% 后才激活 replay。

## 4) Experiments（必须含具体数字）
### 4.1 Experimental setup
- **训练数据：** OpenR1-Math 45K subset。
- **模型：** 5 个骨干模型——Qwen2.5-Math 7B/1.5B（Base）、Qwen2.5-7B Instruct、Llama-3.1 8B（Base/Instruct）。
- **训练配置：**
  - Rollout batch size **128**，update batch size **64**。
  - On-policy 采 **K=8** 轨迹/题；replay 采 **7** 轨迹/题。
  - 经验混合比 **ρ=50%**。
  - Delayed start：batch Pass@1 > **35%** 激活（Llama 除外）。
  - Policy shaping：β=**0.1**。
  - 硬件：**8×H100**。
  - 框架：verl；Reward model：Math-Verify。
- **评测：**
  - ID（6 个）：AIME24, AIME25, AMC, MATH-500, Minerva, OlympiadBench。
  - OOD（3 个）：ARC-c, GPQA-Diamond, MMLU-Pro。
  - AIME/AMC 报 **Avg@32**；其余报 **Pass@1**。
  - Temperature=**0.6**，top-p=**1.0**。

### 4.3 Analysis experiments（强制"现象+解释"）
- **现象1：** Llama-3.1 8B 上 on-policy RLVR 出现 entropy 爆炸和 reward 崩溃，ExGRPO 成功稳定训练。
  **解释（作者）：** 弱模型能力不足导致大多数题 reward=0，探索信号消失；replay 让模型复用早期偶然正确的轨迹，维持有意义的梯度信号。
  **【标注】（我的判断）：** 这个现象对 agent 场景尤为重要——agent 在新环境初期常面临类似"冷启动崩溃"，experience replay 是天然的解药。

- **现象2：** ρ=75%（过度 replay）导致性能低于 on-policy baseline；ρ=25%（过少 replay）则 retired set 增长缓慢。ρ=50% 最优。
  **解释（作者）：** 过高 replay 抑制探索，过低则 replay 信号不足；需要 exploration-exploitation 平衡。
  **【标注】（我的判断）：** 直接启发 agent memory 系统设计：memory replay 比例不是越大越好，需要自适应调节。

- **现象3：** 去掉 question selection 后，retired set 快速增长（因为大量简单题反复被抽到并全部答对），buffer 萎缩，性能下降。
  **解释（作者）：** 没有难度感知的采样会导致"过度消费简单经验"，与 curriculum learning 精神一致。
  **【标注】（我的判断）：** 这就是 memory 的 precision-recall 权衡：对 agent 来说，记忆库的"难度路由"比纯粹扩容更关键。

- **现象4：** 选最高熵轨迹（反向消融）导致性能下降，验证低熵 ≈ 高 CoT 质量的假设。
  **解释（作者）：** 高熵轨迹可能是"lucky hit"——答案碰巧对但推理链无效，replay 这些会导致 snowball effect。
  **【标注】（我的判断）：** 对 agent memory 的直接启发：不能只看 outcome reward，要有推理质量过滤（类似 confidence gate / process reward）。

## 5) Why it matters for our work
- **Agentic memory：** ExGRPO 的 bucketed replay + entropy-based selection 思路可直接迁移到 agent 的 experience memory 管理——按任务难度分桶、按推理质量过滤、设 retired set 避免过拟合已解决问题。
- **Long-context reasoning：** 虽然本文聚焦 math，但"中等难度最优"的发现暗示 long-context task 也应优先训练"模型刚好能部分解决"的样本，而非极难或极简的。
- **Multi-modal agents + RL：** 作者明确将 multi-modal reasoning 和 agentic RL 列为 future work；ExGRPO 的框架（experience management + mixed-policy optimization）天然可扩展到多模态 agent 的 RLVR 训练。
- **Retrieval-augmented reasoning：** entropy 作为在线质量代理的发现，可用于 RAR 系统中对检索到的 CoT 做质量过滤。

### Case 对齐（>=2）
- **Case A（对齐 `2026-03-19_arise.md`）：** 都关注"如何在 RL 训练中有效利用经验"，但 ARISE 走 skill library（高层结构化经验），ExGRPO 走 trajectory replay（低层原始经验）；两者互补——未来可以做"先粗筛 trajectory → 再抽象为 skill"的两级 pipeline。
- **Case B（对齐 `2026-03-11_evo-memory.md`）：** Evo-Memory 侧重 test-time memory evolution，ExGRPO 侧重 training-time experience management；两者共同印证"经验不是越多越好，质量管控是关键"。

## 6) Actionable next step
- [ ] 在现有 RLVR pipeline 中加入 correctness-based bucketing + 低熵轨迹选择，对比 vanilla replay 和 ExGRPO-style replay 的训练稳定性和收敛速度。
- [ ] 验证 entropy 作为 CoT 质量代理是否也适用于多模态任务（图文推理、工具调用），尤其在 agent 场景下。
- [ ] 探索 Retired Set 机制在 agent memory 中的对应物：已完全掌握的策略/技能自动下沉到冷存储，训练/推理聚焦未解决问题。

## 7) 评分解释（必填）
- **质量分 2/2：** 方法设计有完整的预分析支撑，实验覆盖 5 个模型 × 9 个 benchmark，消融充分（question selection / trajectory selection / policy shaping / ρ 比例），且提供了完整的训练动态分析和 buffer/retired set 演化可视化。
- **Observation 分 1/2：** "中等难度 + 低熵"的经验选择原则是有价值的发现，但主要证据仍来自 math reasoning 单一 domain；entropy 作为 CoT 质量代理的泛化性（到多模态/agent 场景）还需更多验证。
- **总分 4/5：** 基础 1 + 质量 2 + Observation 1。
- **为什么不是更高分：** 核心方法是 experience replay + 优先采样的经典组合在 LRM 场景的适配，novelty 相对有限；entropy-based selection 的理论解释偏 empirical；跨任务/跨模态验证不足。
