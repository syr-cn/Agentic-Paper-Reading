## 1) 一句话 Why-read
这篇的核心价值是把“轨迹记忆”升级为“可检索、可演化、可被策略内化的技能库（SkillBank）”，并在 RL 训练中递归更新，直接回应 agent memory 的三个痛点：噪声、上下文膨胀、静态不可进化。

---

## 2) CRGP

### C — Context
LLM agent 在复杂任务上表现强，但大多数执行是“单回合孤立决策”；传统 memory 往往保存原始轨迹，导致 token 冗余高、噪声重、迁移弱。

### R — Related Work
论文对比了四类方法：
1. **Prompt/Memory**：ReAct, Reflexion, Mem0, ExpeL, MemP, SimpleMem。
2. **RL-only**：RLOO, GRPO。
3. **Memory+RL**：MemRL, EvolveR, Mem0+GRPO, SimpleMem+GRPO。
4. **Closed-source LLM**：GPT-4o, Gemini-2.5-Pro。

### G — Research Gap
作者给出三个关键 gap：
1. **表示层 gap**：原始轨迹难提炼高层可复用原则；
2. **训练层 gap**：很多方法只在检索侧“加记忆”，未把能力内化进策略参数；
3. **生命周期 gap**：记忆库通常静态，难随策略升级而共演化。

### P — Proposal
SkillRL 三段式：
1. **Experience-based distillation**：成功轨迹提炼成功策略，失败轨迹提炼失败教训；
2. **Hierarchical SkillBank**：通用技能（总是带）+ 任务专属技能（语义检索 Top-K）；
3. **Recursive evolution in RL**：每次验证后针对低成功率类别增补/修订技能库。

## 4) Experiments

### 4.1 Experimental setup（可提取数字）
- **Base model:** Qwen2.5-7B-Instruct
- **Teacher:** OpenAI o3（技能蒸馏 + SFT 数据）
- **RL:** GRPO
- **Learning rate:** 1e-6
- **Batch size:** 16
- **Group size (G):** 8
- **Gradient accumulation:** 4
- **Task-specific retrieval K:** 6
- **Threshold δ:** 0.4（触发失败类别演化）
- **Benchmarks:** ALFWorld、WebShop、7 个 search-augmented QA（NQ/TriviaQA/PopQA/HotpotQA/2Wiki/MuSiQue/Bamboogle）

### 4.2 Main result table（核心数字）

#### A) ALFWorld + WebShop（Table 1）
| Setting | Baseline | SkillRL | Delta |
|---|---:|---:|---:|
| ALFWorld All（vs GRPO） | 77.6 | 89.9 | **+12.3** |
| WebShop Success（vs GRPO） | 66.1 | 72.7 | **+6.6** |
| ALFWorld Cool（vs GRPO） | 72.5 | 95.5 | **+23.0** |
| ALFWorld Pick2（vs GRPO） | 64.7 | 87.5 | **+22.8** |
| ALFWorld All（vs Mem0+GRPO） | 54.7 | 89.9 | **+35.2** |
| WebShop Success（vs Mem0+GRPO） | 37.5 | 72.7 | **+35.2** |
| ALFWorld All（vs GPT-4o） | 48.0 | 89.9 | **+41.9** |
| ALFWorld All（vs Gemini-2.5-Pro） | 60.3 | 89.9 | **+29.6** |

补充：SkillRL 在 Table 1 原始分项里，ALFWorld 六子任务分别为 **Pick 97.9 / Look 71.4 / Clean 90.0 / Heat 90.0 / Cool 95.5 / Pick2 87.5**；WebShop **Score 85.2 / Succ 72.7**。

| Setting | Baseline | SkillRL | Delta |
|---|---:|---:|---:|
| 平均分（vs Search-R1） | 38.5 | 47.1 | **+8.6** |
| 平均分（vs EvolveR） | 43.1 | 47.1 | **+4.0** |
| Bamboogle（vs EvolveR） | — | — | **+19.4** |

#### C) Ablation（Table 3，正文给出的幅度）
- 去掉层级结构（仅 task-specific）：ALFWorld **-13.1%**，WebShop **-11.3%**。
- 用 raw trajectory 替代技能库：性能下降 **最高约 25%**。
- 去掉 cold-start SFT：约 **-20%**。
- 去掉动态演化：约 **-5.5%**。

### 4.3 Analysis（现象 + 解释）
1) **现象：** SkillRL 对纯 GRPO 提升明显（ALFWorld 89.9 vs 77.6；WebShop 72.7 vs 66.1）。
   **解释：** 技能抽象把高噪轨迹变成高密策略，并通过 SFT+RL 内化到策略，而非仅做 prompt 贴片。

2) **现象：** 困难子任务增益更大（Cool +23.0，Pick2 +22.8）。
   **解释：** 层级技能更适合多步规划、状态跟踪与失败规避，收益集中在 long-horizon 场景。

3) **现象：** 递归演化显著加速训练（60 steps 过 80%，无演化约 90 steps 且峰值更低）。
   **解释：** 每轮失败类别回流补技能，构成“失败模式→策略补丁”的正反馈闭环。

4) **现象：** 上下文长度下降约 10.3%（约 1450 → <1300）。
   **解释：** 蒸馏把冗余操作与试探过程移除，仅保留可执行原则，缓解 context bloat。

5) **现象：** 技能库扩张以任务专属为主（43→80），通用技能增长较缓（12→20）。
   **解释：** 【标注】随着策略提升，新增失败模式更常出现在细粒度任务边界，促使专属策略快速细化；但长期需做去重/淘汰防膨胀。

## 5) Why it matters for our work
1. **Memory 对象升级**：应从“片段轨迹”转为结构化 `skill_card`（name/principle/when_to_apply/anti-pattern）。
2. **Long-context 可控化**：Skill abstraction 提供“性能提升 + token 下降”的双重收益。
3. **RL 闭环可迁移**：失败回放→技能增补→策略更新，可直接迁到网页/工具/多模态 agent。
