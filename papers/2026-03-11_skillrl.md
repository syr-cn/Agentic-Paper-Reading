## 1) 一句话 Why-read
这篇工作把“轨迹记忆”升级成“可检索、可演化的技能库”，并且在 RL 训练中动态更新，明显击中了 agent memory 的核心痛点（噪声、上下文膨胀、不可泛化）。

---

## 2) CRGP

### C — Context
LLM agents 在复杂任务上强，但多数是“每次任务从零开始”；已有 memory 方法多存原始轨迹，导致冗余高、噪声大、token 成本高，且难以转化为可迁移策略。

### R — Related Work
作者对比了四类：
1. **Prompt/Memory methods**：ReAct, Reflexion, Mem0, ExpeL, MemP, SimpleMem。
2. **RL-only methods**：RLOO, GRPO。
3. **Memory+RL methods**：MemRL, EvolveR, Mem0+GRPO, SimpleMem+GRPO。
4. **Closed-source LLMs**：GPT-4o, Gemini-2.5-Pro。

### G — Research Gap
论文明确指出三个 gap：
1. **表示层 gap**：原始轨迹难以抽象出高层可复用知识；
2. **训练层 gap**：很多方法只在检索侧“贴记忆”，没有把记忆能力内化到策略；
3. **生命周期 gap**：记忆/技能通常是静态库，无法随策略改进同步演化。

### P — Proposal
SkillRL 三段式：
1. **Experience-based distillation**：成功轨迹提炼策略，失败轨迹提炼“失败教训”；
2. **Hierarchical SkillBank**：通用技能 + 任务专属技能，检索时通用全带，专属 Top-K；
3. **Recursive evolution in RL**：每个验证周期按失败类别扩增/修订技能，使技能库和策略共演化。

### 4.3 Analysis（现象 + 解释 + 我的判断）

1) **现象：** 用 SkillBank 后，ALFWorld/WebShop 明显优于纯 GRPO（89.9 vs 77.6；72.7 vs 66.1）。  
   **解释（作者）：** 技能抽象把冗余轨迹压成高密度策略，并通过检索在决策时给到“可执行启发”。  
   **我的判断：** 这不是简单“加提示词”，因为作者做了 cold-start SFT + RL 内化，属于“表示+训练”双改；提升可信。

2) **现象：** 在复杂子任务（Cool/Pick2）增益更大（+23.0/+22.8）。  
   **解释（作者）：** 层级技能能更好支持多步规划、状态跟踪和失败规避。  
   **我的判断：** 增益集中在高难子任务，说明方法价值主要在 long-horizon credit assignment，而非简单任务的提示增强。

3) **现象：** 递归演化使收敛加速（60 step >80%，无演化约 90 step 且上限更低）。  
   **解释（作者）：** 每轮从验证失败中补技能，持续填补策略盲区，形成正反馈。  
   **我的判断：** 这是“在线课程学习”的一种实现；关键风险在于 teacher 生成技能可能引入幻觉规则，需后续做技能质量门控。

4) **现象：** 上下文更省：平均 token 从约 1450 降到 <1300（约 -10.3%）。  
   **解释（作者）：** 10–20× 轨迹压缩将噪声移除，仅保留可复用策略要点。  
   **我的判断：** 这点对长上下文 agent 特别关键；但 -10.3% 的在线 prompt 降幅相对温和，说明部署收益还受检索模板和推理链长度影响。

5) **现象：** 技能库规模随训练增长（55→100），且以任务专属技能增长为主（43→80）。  
   **解释（作者）：** 策略变强后探索到新失败模式，推动专属技能细化。  
   **我的判断：** 这支持“技能库应是动态资产”；但也提示未来要做技能去重/衰减，否则长期可能出现库膨胀和检索污染。

## 5) Why it matters for our work
1. **对 agent memory：** 这篇证明“轨迹记忆对象”应从原始片段升级为结构化 skill card（name/principle/when_to_apply）。
2. **对 long-context：** 技能化能在不牺牲成功率的前提下降低 token 压力，是上下文预算管理的可行路线。
3. **对 multimodal RL：** “失败轨迹→失败教训→技能补丁”的闭环可迁移到视觉/网页/工具调用场景（把失败模式作为跨模态策略单元）。

## 6) Actionable next steps（3 条，可执行）

1. **先做 memory schema 升级（1 周内可落地）**  
   在现有 agent 中新增 `skill_card` 结构：`{name, principle, when_to_apply, anti_pattern, source_episode_ids}`；并把原 trajectory memory 仅作为离线素材，不直接在线拼接。

2. **做一次 long-context A/B（两组各 ≥200 任务）**  
   A 组：raw memory 检索；B 组：skill 检索（通用全带 + TopK 专属，K=6 起步）。统一模型与预算，记录 success、prompt tokens、latency、失败类型分布。

3. **做 multimodal RL 小闭环（网页/视觉代理）**  
   在一个多步视觉任务（如网页购物/GUI 操作）上实现“每个验证周期失败回放→技能增量更新→下一轮 RL”，并加两道门控：
   - 规则冲突检测（新旧技能矛盾打分）
   - 低价值技能淘汰（长期不命中 + 无增益）
