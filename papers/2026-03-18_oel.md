## 1) 一句话 Why-read（必填）
- **Key claim/contribution + key observation：** OEL把“部署时交互轨迹”变成“可训练知识”，并通过**经验抽取→on-policy 蒸馏**形成可迭代闭环；关键观察是**抽取知识显著优于 raw trajectory**，且 on-policy 一致性直接影响收益。

---

## 2) CRGP 拆解 Introduction（必填）
### C — Context
- 主流大模型优化仍以离线为主（SFT/RL），部署后的真实交互经验基本被浪费。
- 真实环境通常只返回文本反馈，不提供可直接用于RL的标量奖励；同时服务端训练端往往无法访问用户侧环境。

### R — Related work
- 论文把自己放在两条线的交叉：
  1) 从部署经验学习（learning from experience）
  2) context distillation / on-policy distillation
- 目标不是再造 reward model，而是直接利用文本反馈构造可学习信号。

### G — Research gap
- 现有实践缺少“**无奖励、无环境访问**”条件下可持续在线更新模型的闭环。
- 直接使用 raw trajectory 信号质量不足，难稳定内化。

### P — Proposal
- OEL 两阶段：
  1) **Extraction**：从多轮轨迹中抽取可迁移经验知识并累积；
  2) **Consolidation**：on-policy context distillation，把知识内化到参数，推理时不再依赖知识上下文。
- 两阶段反复迭代，形成 online learning loop。

## 4) Experiments（必须含具体数字）

### 4.1 Experimental setup
- **任务/数据：** Frozen Lake、Sokoban（TextArena）。
- **模型配置：** Qwen3-1.7B / Qwen3-4B / Qwen3-8B（thinking）与 Qwen3-4B-Instruct-2507（non-thinking）。
- **Extraction stage（明确数字）：**
  - structured：`n ∈ {25, 50}`，`L_max=8192`
  - unstructured：`n=15`，`L_max=2048`
  - 累积重复次数：`K=10`（不同随机种子）
- **Consolidation stage（明确数字）：**
  - 每轮训练步数：`20 或 100`
  - 每步 game samples：`64`
  - 每轮轨迹需求：`1280 或 6400`
  - 每次环境交互最多 `5 turns`
  - 每 turn 最大响应 `1024 tokens`
  - reverse KL 计算采用 student 概率最高的 `top-256` 词表项
  - 学习率搜索区间：`[1e-6, 5e-6]`
  - 采样温度：`0.7`
- **评测：**
  - pass rate（ID）
  - OOD：IF-Eval prompt-level strict accuracy
  - 测试集：`size-128` game maps，结果平均 `10` random seeds
- **Table 4 最终超参（明确数字）：**
  - Qwen3-1.7B/FrozenLake：Unstructured，n=15，Lmax=2048，lr=5e-6，steps=20
  - Qwen3-4B/FrozenLake：Structured，n=25，Lmax=8192，lr=1e-6，steps=20
  - Qwen3-8B/FrozenLake：Structured，n=50，Lmax=8192，lr=1e-6，steps=100
  - Qwen3-4B-Instruct/Sokoban：Structured，n=50，Lmax=8192，lr=1e-6，steps=100

### 4.3 Analysis experiments（>=3，现象+解释）
1. **现象：** Sokoban 上 raw trajectory consolidate 仅 `7.8`，几乎贴近无经验 `7.5`；而抽取知识 consolidate 到 `21.4`。  
   **解释（作者）：** raw 轨迹噪声/冗余高，难被参数稳定吸收；抽取后的经验更可学习。  
   **【标注】（我的判断）：** 这是本文最硬证据，说明“信息可学习性”比“原始信息量”更关键。

2. **现象：** Frozen Lake 上 self knowledge（`23.8/31.1`）显著优于来自更大模型4B的 knowledge（`18.0/22.7`）。  
   **解释（作者）：** on-policy 一致性更好，知识分布与当前策略匹配更强。  
   **【标注】（我的判断）：** 对 memory 系统很关键：并非教师越强越好，分布一致常优先。

3. **现象：** 响应长度降至初始约 `70%`，同时主任务通过率上升（图1/图5趋势）。  
   **解释（作者）：** 经验被内化后，模型可更快到达有效动作，减少无效推理。  
   **【标注】（我的判断）：** 若在开放任务上复现，这意味着准确率、时延、token成本可三赢。

4. **现象：** on-policy 蒸馏相比 off-policy 更能保持 OOD（IF-Eval）并缓解遗忘（图6趋势）。  
   **解释（作者）：** on-policy 训练目标与当前策略分布一致，减少偏移带来的能力塌陷。  
   **【标注】（我的判断）：** 对部署型持续学习很实用，但仍需更多绝对分数佐证。

### 4.4 Case（>=2）
- **Case 1（正例：抽取知识远胜 raw 轨迹）**  
  场景：Sokoban，Qwen3-4B-Instruct-2507。  
  证据：raw consolidate=`7.8` vs knowledge consolidate=`21.4`（+13.6）。  
  含义：若直接把日志喂训练，收益可能接近零；必须经过经验抽取。

- **Case 2（正例：self/on-policy 知识优于外部强模型知识）**  
  场景：Frozen Lake，policy 为 Qwen3-1.7B。  
  证据：self consolidate=`31.1` > 4B consolidate=`22.7`；self in-context=`23.8` > 4B=`18.0`。  
  含义：知识源与策略模型分布一致性，直接决定蒸馏效果上限。

- **Case 3（效率例：知识内化降低长度）**  
  场景：Frozen Lake 多轮 OEL。  
  证据：平均响应长度约降到 `70%`。  
  含义：OEL 不仅提分，还可能带来可观推理成本收益。

## 5) Why it matters for our work
- 对 **agent memory**：可以把“日志存档”升级为“经验抽取 + 参数内化”的双层记忆闭环。
- 对 **long-context**：不是无限加上下文，而是把稳定经验迁入参数，减少推理时上下文负担。
- 对 **multimodal RL/agent**：on-policy 一致性结论可迁移到多模态经验蒸馏与技能学习。
