## 1) 一句话 Why-read
XSkill 的核心价值是把“可复用知识”明确拆成 **经验（action-level）+技能（task-level）** 两条流，并且在**视觉上下文中抽取/检索/改写**，实证上在多基座、多 benchmark 持续优于 tool-only 与已有 memory baseline。

---

## 2) CRGP

### C — Context
多模态 agent 已能调工具，但仍有两类瓶颈：
1. **工具使用低效**（简单题步数冗余、复杂题探索深度不足）；
2. **工具编排僵化**（单路径执行、跨任务泛化差）。

作者目标是：在**不更新模型参数**前提下，让 agent 能从历史轨迹持续改进。

### R — Related work
作者把相关工作分两脉：
- **Experience memory**：偏战术、局部决策（何时选哪种工具，怎么恢复错误）；
- **Skill/workflow memory**：偏程序化、高层流程（步骤模板、工具链编排）。

已有方法的共同短板：大量依赖文本日志，弱化视觉 grounding；且多局限于 GUI/空间推理等窄场景。

### G — Research gap
本文要填的空白很明确：
1. 过去少有框架统一建模“经验+技能”双流；
2. 多模态场景下，知识抽取与检索缺少视觉对齐；
3. 缺少从 accumulation→inference→再反馈的闭环设计。

### P — Proposal
提出 **XSkill**：
- **知识结构**：Skill Library（Markdown）+ Experience Bank（JSON）；
- **Phase I Accumulation**：多路径 rollout → 视觉 grounding 总结 → 跨 rollout critique → 分层合并；
- **Phase II Inference**：任务分解检索 → 视觉上下文改写 experience → 适配 skill 后注入系统提示；
- **闭环**：记录 usage history 回流到下一轮 accumulation。

## 4) Experiments

### 4.1 Experimental setup（含具体数字/设置）

#### 数据与任务
- **5 个 benchmark**，覆盖 3 类域：
  1) visual tool use：VisualToolBench、TIR-Bench  
  2) multimodal search：MMSearch-Plus、MMBrowseComp  
  3) 综合困难场景：AgentVista
- 每个 benchmark：随机抽 **100 tasks** 做训练集用于 accumulation，其余做测试。

#### Baselines
- AWM、Dynamic CheatSheet（DC）、Agent-KB，以及 w/ tools（tool-only）。

#### 工具集
- code interpreter、web search、image search、visit（按数据集激活子集）。

#### 指标
- 每题 **N=4 rollouts**；报告：
  - **Average@4**（4 次成功率均值）
  - **Pass@4**（4 次中至少一次成功）

#### 模型与实现
- 主实验 backbone：Gemini-2.5-Pro、Gemini-3-Flash、GPT-5-mini、o4-mini。  
- 检索 embedding：**text-embedding-3-small**。  
- 检索参数：**top-k = 3**。  
- 生成参数：**temperature=0.6, top-p=1.0, max turns=20**。  
- 迁移设定：GPT-5-mini 与 o4-mini 直接使用 Gemini-3-Flash 累积知识（测试 cross-model transfer）。

#### 方法中的符号阈值（是否给出具体值）
- 原文定义了 `L_max^e`, `L_max^K`, `θ_sim`, `τ_min`, `n_g`, `N_max^E` 等。  
- **具体数值：原文当前可提取片段未给出**（需查 appendix 才可能补全）。

### 4.3 Analysis（至少 3 条：现象 + 解释 + 我的判断）

#### 1) 现象：双流都重要，但 skill drop 更大
- **现象**：Ablation 中去掉 experiences 掉 **3.04** 点；去掉 skills 掉 **3.85** 点。  
- **解释（作者）**：skills 更直接降低执行层面的结构性错误，experience 提供策略灵活性；两者互补。  
- **我的判断**：这说明在多工具链任务里，“先把执行稳定性打牢”收益更大；experience 更像在稳定基座上的策略增益器。

#### 2) 现象：Phase I（知识质量）比 Phase II（检索适配）更关键
- **现象**：去掉 Experience/Skill manager（Phase I）分别掉 **4.09/3.62**；去掉 Task Decomposition/Adaptation（Phase II）掉 **1.28/1.52**。  
- **解释（作者）**：知识库质量是上限，检索器只是把已有知识拿出来。  
- **我的判断**：这对 memory-agent 实做很关键：优先资源应放在“高质量写入/合并/去重”，而不是过度堆 retrieval trick。

#### 3) 现象：skills 显著降低执行错误
- **现象**：VisualToolBench 上总体错误率从 **29.9% (168)** 降到 **15.3% (95)**；syntax error **114→71**，tool-name error **16→2**。  
- **解释（作者）**：技能模板约束了调用格式和工作流，减少无效试错。  
- **我的判断**：这非常像“可执行 SOP”对 agent 的作用；若要进一步扩展到长上下文任务，可把 skill 模板加上“失败后回退节点”。

#### 4) 现象：experiences 改变工具选择分布（更战术）
- **现象**：VisualToolBench 中 code interpreter 使用率 **66.63%→74.49%**（full pipeline **76.97%**）；MMSearch-Plus 中 code interpreter **6.18%→13.21%**，image search **15.43%→24.63%**。  
- **解释（作者）**：经验条目提供了条件-动作层面的局部策略，促使工具选择更贴合问题属性。  
- **我的判断**：经验流本质上在学“策略先验”，它不一定提升每一步正确率，但能显著减少策略失配。

#### 5) 现象：rollout 数越大，效果越稳步上涨
- **现象**：在 N∈{1,2,3,4} 下，Average@4 与 Pass@4 都持续上升（Pass@4 更陡）。  
- **解释（作者）**：更多轨迹差异让 critique 能更好识别因果成功因素。  
- **我的判断**：这是“多样性→可学习性”的典型证据；实际系统可考虑在预算固定下做“有分歧优先采样”的 rollout 策略。

## 5) Why it matters for our work

对 agent memory / long-context / multimodal RL 的启发非常直接：
1. **Memory schema**：明确分 action-level experience 与 task-level skill，避免单池记忆语义冲突；
2. **Visual-grounded memory**：检索/query 不能只看文本任务，要把视觉 state 作为一级信号；
3. **Write-path first**：先把 accumulation 质量做好（总结、批判、合并、清洗），再谈复杂 retrieval；
4. **Cross-model KB**：知识外置后具备模型间迁移潜力，适合混合模型栈系统。

## 6) Actionable next steps（3 条，可执行，面向 agent memory/long-context/multimodal RL）

1. **实现双流记忆最小闭环（2 周）**  
   - 做 `experience_bank.jsonl`（condition/action/embedding/source/outcome）+ `skill_library.md`（workflow/template/version）。  
   - 先在 1 个视觉工具任务集跑：对比 single-pool vs dual-stream 的 Average@k、错误率、工具调用熵。

2. **长上下文写入质量实验（1~2 周）**  
   - 在 accumulation 里加入“跨 rollout critique + 相似度合并 + 长度预算裁剪”。  
   - 指标：记忆规模增长曲线、检索命中率、重复条目率、最终 Pass@4。  
   - 重点验证“写入质量提升是否比检索花活更值钱”（对应本文 Phase I>Phase II 结论）。

3. **多模态 RL 式策略蒸馏（原型）**  
   - 用 experience 条目构造轻量 reward shaping：命中高价值经验时给正奖励，触发高频错误模板时惩罚。  
   - 先离线 bandit/偏好学习，不动大模型参数；目标是让工具选择分布向“高成功策略”迁移。  
   - 观察是否能复现本文“experience 改变工具使用结构”的趋势。
